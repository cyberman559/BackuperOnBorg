## ✨ Особенности

- 🔒 Шифрование репозитория borg
- 💾 Поддержка нескольких типов баз данных:
  - 🗄️ MySQL (полная поддержка)
  - 🐘 PostgreSQL (полная поддержка)
  - 🟦 MSSQL (частичная поддержка)
- 🗂️ Умное исключение служебных файлов (кэш, логи, статистика для Bitrix)
- 🕒 Автоматическое управление ретеншеном (7 дней, 4 недели, 6 месяцев)
- 🔌 Поддержка VPN-соединений для удалённого бэкапа
- 🔄 Автоматическое определение времени запуска (с 05:15 ежедневно)

## 📁 Структура проекта

| Файл | Описание |
|------|----------|
| `start.sh` | Основной скрипт запуска резервного копирования |
| `borg.sh` | Скрипт для создания дампов и архивации на удалённом сервере |
| `unborg.sh` | Скрипт для восстановления из архива |
| `adduser.sh` | Скрипт для настройки нового пользователя и инициализации репозитория |
| `*.yaml.example` | Примеры конфигурационных файлов borgmatic для Bitrix, Laravel и 1С |
| `settings.conf.example` | Пример файла настроек проекта |

## 📋 Требования

### На сервере (куда делается бэкап):

- `borg` и `borgmatic`
- SSH-сервер
- Поддерживаемая СУБД (MySQL, PostgreSQL, MSSQL)

### На клиенте (где хранятся репозитории):

- `borg`
- `borgmatic`
- SSH-доступ к серверам

## 🚀 Быстрый старт

1. Склонируйте репозиторий

2. Запустите скрипт настройки нового пользователя (требуются права root):
   ```bash
   ./adduser.sh
   ```

3. Отредактируйте созданные конфигурационные файлы:
   - `/root/.borg/projects/<username>/full.yaml`
   - `/root/.borg/projects/<username>/settings.conf`

4. Запустите бэкап вручную для тестирования:
   ```bash
   ./start.sh <username> force
   ```
